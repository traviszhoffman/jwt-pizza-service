Incident: 2026-04-07

**Summary**

Between 9:43am and 10:57am on 2026-04-07, my order endpoint returned HTTP 500 failures on pizza order creation.

Severity was moderate (SEV-2) for my service because order creation was unavailable during that period. Impact lasted until the recorded fix date at 10:57am 2026-04-07.

Detection

I detected the incident when order creation requests failed with status 500 and logs showed this response body: `{"message":"Failed to fulfill order at factory","followLinkToEndChaos":"https://pizza-factory.cs329.click/api/support/65422c39b95b4d04a4e0792dd473d99e/report/nviw7etm"}`. That signature matched fail chaos behavior. The entire log as recieved: {"authorized":true,"path":"/api/order","method":"POST","statusCode":500,"reqBody":{"franchiseId":1,"storeId":1,"items":[{"menuId":1,"description":"Veggie","price":0.05},{"menuId":1,"description":"Veggie","price":0.05},{"menuId":1,"description":"Veggie","price":0.05},{"menuId":1,"description":"Veggie","price":0.05}]},"resBody":{"message":"Failed to fulfill order at factory","followLinkToEndChaos":"https://pizza-factory.cs329.click/api/support/65422c39b95b4d04a4e0792dd473d99e/report/nviw7etm"}}

To improve detection time, I would lower the alert level I needed for failed order creation as I had my alerts set to high. 

Impact
From 9:43 to 10:57 am, users calling my order API could not create orders and did not receive a JWT.

Impact scope was limited to my vendor integration. No full-system outage occurred.

Response

Root cause

The root cause was intentional fail chaos activation. When active, middleware forces order requests to return 500 until the matching fix code is reported.

Resolution

I restored service by calling the support link returned by the failed request:

https://pizza-factory.cs329.click/api/support/65422c39b95b4d04a4e0792dd473d99e/report/nviw7etm

After that, chaos state changed 'to No chaos currently executing'. Order creation then returned to normal.

Prevention

This type of incident can happen again during future chaos tests. Better alerting and a short runbook for chaos signatures and remediation will reduce mitigation time.

Action items

1. Add alerts for order API creation issues 500S that are lower than what I previously set.
2. Keep a one-page runbook with the exact remediation steps using `reportUrl`.
3. Add a quick post-fix verification checklist (successful order + no new 5xx for 5 minutes).
